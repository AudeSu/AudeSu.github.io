Waar wordt de event listener gekoppeld aan de slider?

De event listener wordt gekoppeld aan de slider in de JavaScript code met de volgende regels:

sliders[0].addEventListener("change", update);
sliders[0].addEventListener("input", update);

Waarom moeten we die op twee soorten events koppelen?

We moeten de event listener op twee soorten events koppelen (change en input), omdat de input event elke verandering van de slider registreert terwijl de change event alleen wordt geactiveerd wanneer de slider losgelaten wordt na een verandering.

In de CSS file wordt nergens een rode kleur opgegeven, waar wordt dan wel de rode kleur van het blokje ingesteld?

De rode kleur van het blokje wordt ingesteld via JavaScript, meer bepaald met de volgende code: colorDemos[0].style.backgroundColor="red";

Waarom schrijven we telkens sliders[0] en colorDemos[0] en niet gewoon sliders of colorDemos?

We schrijven telkens sliders[0] en colorDemos[0] omdat document.getElementsByClassName een array teruggeeft van alle elementen die voldoen aan de opgegeven class. Aangezien we maar één slider en één blokje hebben, weten we dat deze zich op de eerste positie van de array bevinden.

Waar vind je de CSS-properties die programmatorisch werden ingesteld, zoals bv. de rode achtergrondkleur van het blokje?

Je kan de CSS-properties die programmatorisch werden ingesteld bekijken op het "Computed" tabblad van de Chrome Developer Tools.
